A ideia principal é conhecer o funcionamento interno do PREACT. Preact é compatível (a nível de API) com REACT (14 ,15) e tem um código muito menor, facilitando o entendimento e a desmistificação do funcionamento de alguns frameworks modernos.


Preact é biblioteca que implementa a maior parte da API do REACT e que tem apenas ~3kb de tamanho. Já temos o react, pq diabos precisamos do PREACT?
-mais leve
-foco na web
-performance
-internet lenta
-smartphones fracos

Dependendo da internet cada byte importa. E não é gasto tempo somente para fazer download de uma biblioteca, também é necessário fazer o parse, e isso vai impactar no tempo de interação.
Claro que nada disso faz sentido se as devidas precauções não forem tomadas (carregar somente o necessário, lazy load, cache, sw, etc…)
As vezes se torna uma bazuca para matar uma mosca usar react para resolver/implementar uma página simples. Mas e se ela crescer? Vais querer escrever na mão?

O problema que o Preact tenta resolver é trazer o modelo de implementação do REACT para ambientes que precisam ser pequenas (em tamanho de lib) e carregar rápido


Espero que no fim vcs tenham uma nova blbioteca fácil e usar, e saibam pelo menos um pouco do funcionamento interno.
